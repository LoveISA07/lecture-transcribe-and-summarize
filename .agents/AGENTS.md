# 專案規則：自動化薈萃分析流程

- **前置校正與 Metadata 自動生成**：在對新音訊執行 `generate_final_transcript.py` 之前，Agent **必須主動分析原始 SRT 內容中的 ASR 辨識錯字、專有名詞與口誤**，並自動於同目錄下生成 `<主題>_metadata.json`（包含預估章節時間軸、錯字校正對照表、關鍵字）。**嚴禁**在未生成專屬 Metadata 的情況下直接執行無校正的標點還原。

- **LLM 前後文二次校正**：在 `generate_final_transcript.py` 生成初步逐字稿後，Agent **必須主動讀取該檔案，並利用大模型的理解能力進行前後文二次校正**，直接修改逐字稿中的語意不通與殘留錯字（如 `郭忠祐` ➡️ `血糖波動`），以確保作為薈萃基礎的逐字稿達到最優品質。

- **自動化執行**：當課程逐字稿（`主題-逐字稿.md`）經二次校正生成完畢後，Agent 必須**主動且立即**啟動內容薈萃整理，自動生成並寫入以下兩份檔案，不需等待使用者下達後續指令：
  1. `主題-書面知識報告.md`（教科書級知識報告，含 Q&A）
  2. `主題-重點整理.md`（含 Mermaid 邏輯地圖、章節重點、詞彙表與錯字對照）
